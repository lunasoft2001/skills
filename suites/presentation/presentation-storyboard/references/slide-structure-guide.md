# Slide Structure Guide

Reference models and visual suggestion catalog for `presentation-storyboard`.

---

## Narrative Arc Models

### Standard Business Presentation (default)

```
Act 1 — Context (20%)     → Why this matters / the problem / the opportunity
Act 2 — Content (60%)     → Evidence, analysis, proposal, story
Act 3 — Action (20%)      → Recommendations, next steps, call to action
```

### Pitch Deck (for investors/partners)

```
1. Hook slide (problem statement — 1 sentence)
2. Market size (TAM/SAM/SOM)
3. Solution (product demo or concept)
4. Traction / proof (metrics, customers, ARR)
5. Business model (how you make money)
6. Team (founders, key hires)
7. Ask (funding amount, use of funds)
```

### Educational / Workshop

```
1. Agenda + learning objectives
2. Context / background
3. Core concept (chunked into 2-4 slides)
4. Examples / case studies
5. Practice or exercise prompt
6. Key takeaways + resources
```

### Executive Summary (10 min or less)

```
1. Situation (1 slide — the key problem or decision)
2. Findings (2-3 slides — evidence)
3. Recommendation (1 slide — clear action)
4. Next steps (1 slide — owners + dates)
```

## Visual Suggestion Catalog

Match the visual type to the content type — never suggest a visual that misrepresents the data.

| Content type | Recommended visual |
|---|---|
| Single key number / KPI | Large bold statistic, minimal slide |
| Trend over time | Line chart |
| Comparison between categories | Bar or column chart |
| Part-to-whole / share | Pie or donut chart (max 5 slices) |
| Ranking | Horizontal bar chart |
| Relationship / correlation | Scatter plot |
| Process or workflow | Flowchart / process diagram |
| Org structure / hierarchy | Org chart or tree diagram |
| Geographic distribution | Map visualization |
| Qualitative insight | Quote callout or text-heavy card |
| Matrix / options comparison | 2×2 matrix |
| Timeline | Horizontal timeline |
| Complex system | Architecture or concept diagram |
| Executive recommendation | Bold headline + 3 bullets max |

## One-Message-Per-Slide Rule

Each slide must pass this test:
> "If I can only tell the audience one thing from this slide, what is it?"

That answer is the slide's **message** (the headline). Everything else is supporting evidence.

Fail patterns to avoid:
- Slide titled "Results" with 4 different conclusions → split into 4 slides
- Slide with a chart AND a table AND bullet points → pick one visual element
- Slide message that says "various factors" → you haven't decided what matters

---

## Transition Phrase Patterns

Good transition phrases link the current slide's conclusion to the next slide's question:

- Problem → Solution: *"So how do we fix this? Let me show you what we built."*
- Data → Insight: *"And what does this tell us? The trend points to one clear pattern."*
- Insight → Action: *"Given this, there's one decision we need to make today."*
- Section break: *"That covers our analysis. Now let's turn to our recommendations."*
