# Obsidian Task Conventions

Normalize operational tasks in Markdown.

## Syntax

- Open task: `- [ ]`.
- Closed task: `- [x]`.

## Priority pattern

- `- [ ] P1 - Critical task`
- `- [ ] P2 - Important task`
- `- [ ] P3 - Backlog task`

## Suggested sections

1. `## Tareas realizadas`
2. `## Tareas en curso`
3. `## Tareas pendientes`
4. `## Backlog`

## Rule

If the note is operational, include at least one checkbox block.
